# PRD-053 Missing BOM Remediation Plan

## Goal

Make missing BOM data-quality issues actionable by deep-linking directly into the Admin Data BOM editor with enough context to create the missing BOM and immediately re-run demand.

## Scope

- Generate missing BOM remediation routes from data-quality issues with `dishId`, `serviceDate`, optional `customerId`, and `scope`.
- Use the report query service date as the missing-BOM effective-date context instead of always using today's date.
- Let Admin Data initialize the adjustments/BOM view from remediation query parameters.
- Let in-page cleanup links switch directly to the BOM editor for the affected dish.
- Show remediation context and a re-run demand action beside the BOM form.
- Preserve existing data-quality table and BOM editor behavior for non-remediation use.
