<objective>
Compose the bounded parser and pure classifier into one Admin-only, full-dataset preview that writes no domain data, issues an immutable actor-bound manifest with deterministic hash/fingerprint/TTL, detects file/parameter/policy/database drift, and closes Gate B without exposing an apply or destructive endpoint.
</objective>

<execution_context>
- Final Phase 3 Gate A, 04-01 parser/security tests and 04-02 classifier/evidence tests must pass before this plan.

- No apply/commit/destructive endpoint, domain `SaveChanges`, cleanup DML, audit insertion, cache invalidation, downstream regeneration, frontend/RTK/UI or old-contract removal is permitted.
</execution_context>

<must_haves>
- D-04-04: preview runs from stable `AsNoTracking` projections, has no Added/Modified/Deleted entries, invokes no `SaveChanges`, and keeps ordered counts/checksums identical.
- D-04-05: immutable manifest contains unguessable preview ID, SHA-256, contract/policy versions, effective date, server actor/scope, deterministic DB fingerprint, issued/expires timestamps, source trace and full action/blocker/history-kept counts.
- D-04-06: file hash, effective date, contract/policy version, actor/scope, expiry or DB fingerprint drift returns stale/conflict and requires a new preview; Phase 5 must reparse/reclassify server-side.
- D-04-08: the controller is class-level `AdminAccess`; actor comes only from `ICurrentUserService`, and client actor/path/candidate/hash/count fields have no authority.
- D-04-09 and D-04-10: multipart content is rejected through the bounded reader and safe response mapping; no server filesystem-path request or leak exists.
- D-04-11: compatibility writer/template/UI remains; route inventory proves there is no canonical apply/destructive route before Gate B.
- Canonical reconciliation counts remain `Unchanged/Create/Version`; legacy counts remain `keep/archive/deactivate/regenerate/delete/block`, with separate typed fields and totals that reconcile to the full dataset.
- The endpoint applies exact pre-model-binding `RequestSizeLimit(26214400)` and `RequestFormLimits(MultipartBodyLengthLimit=26214400)`; a 26214401-byte request never invokes parser or preview service.
</must_haves>
